myFlix

This is a REST API for an application called “myFlix” that interacts with a database that stores data about different movies. The REST API will be accessed via commonly used HTTP methods like GET and POST.

Business Logic URL HTTP method Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user Return data about a genre (description) by name/title (e.g., “Thriller”) /movies/[title] GET None A JSON object holding data about a single movie, containing an ID, title, description, genre, director, imageURL (if featured). Example:
{
id : 1,
name : "The Shawshank Redemption",
description : "Two imprisoned men bond over a number of years, finding solace and eventual redemption through acts of common decency.",
genre : "Drama",
director : "Frank Darabont",
imageURL : "https://www.imdb.com/title/tt0111161/"
}
Return data about a director (bio, birth year, death year) by name /directors/[name] GET None A JSON object holding data about a single director. Example:
{
id : 1,
name : "Frank Darabont",
birth year : 1959,
death year :
}
Return data about a genre (description) by name (e.g. "Comedy") /genres/[name] GET None A JSON object holding data about a single genre. Example:
{
id : 1,
name : "Drama",
description : "In film and television, drama is a category of narrative fiction (or semi-fiction) intended to be more serious than humorous in tone. (Drama (film and television), Wikipedia)"
}
Allow new users to register /users POST A JSON object holding data about the user to add, structure like:
{
name : "Anna Hufland",
username: "spatzi",
email : "annahufland@gmail.com"
} 		A JSON object holding data about the user that was added, including an ID:
{
id : 5324,
name : "Anna Hufland",
username: "spatzi",
email : "annahufland@gmail.com"
}
Allow users to update their user info (username) /users/[username] PUT None A text message indicating that the user info (username) was updated
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/[username]/favorites POST A JSON object holding the name of a favorite movie, structure like:
{
title : "The Dark Knight"
} 		A JSON object holding data about the movie that was added, including an ID:
{
id : 123,
title : "The Dark Knight"
}
Allow users to remove a movie from their list of favorites /users/[username]/favorites/[ID] DELETE None A text message indicating whether the movie was successfully removed
Allow existing users to deregister /users/[username] DELETE None A text message indicating whether the user was successfully removed
